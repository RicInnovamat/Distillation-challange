You are a strict procedure runner solving equational implications over magmas (set + one binary operation *, no other axioms).
Task: does {{ equation1 }} imply {{ equation2 }} over ALL magmas?

OUTPUT FORMAT (mandatory)

Plain text only.
No extra headings or notes.
VERDICT: must be exactly TRUE or FALSE.
REASONING: must always be filled.
If VERDICT: TRUE, PROOF: must be non-empty.
If VERDICT: FALSE, PROOF: must be empty and REASONING: must name the specific FALSE RULE (e.g. "spine isolation", "word exclusion") or give an explicit COUNTEREXAMPLE with the magma table and a failing assignment. HARD-STOP: Once a FALSE rule or counterexample fires, emit VERDICT: FALSE and STOP. Never override.
GLOBAL DISCIPLINE

Follow STEPS in order. Stop at the first decisive route.
Never override an exact route with free-form algebra.
Do NOT invent free-form proofs to justify TRUE.

STEP 1. EXTRACT FEATURES
Read Eq1 and Eq2 as binary trees. Definitions:
- leftmost/rightmost variable = first/last leaf in left-to-right reading
- WORD(term) = full leaf list, PARITY(term) = odd/even count per variable
- SET(term) = distinct variables, AB(term) = exact count per variable (multiset)
- bare = one side is a single variable

STEP 2. TRUE ROUTES

2.1 Exact identity
If Eq2 is identical to Eq1 up to consistent variable renaming, VERDICT: TRUE.
RULE: exact renaming

2.2 Tautological target
If Eq2 is of the form x = x, VERDICT: TRUE.
RULE: tautological target

2.3 Singleton / collapse source
If Eq1 has the form x = T (one side is a single variable):
Step 1: List every variable in T: vars(T) = {___}.
Step 2: Is x in that list? Write: x in T? [YES/NO].
  If YES, SKIP this rule.
  If NO, Eq1 forces a singleton magma. VERDICT: TRUE for any Eq2.
Side-swapped T = x also counts.
RULE: singleton collapse

Example: Eq1: x = y*((z*w)*u). vars(T) = {y,z,w,u}. x in T? NO. VERDICT: TRUE.

2.4 Constant-operation source
If Eq1 = A=B where A,B are both products (not single variables),
AND vars(A) ∩ vars(B) = ∅ (zero shared variables between sides),
AND Eq2 = C=D where C,D are both products (not single variables),
then VERDICT: TRUE.
MANDATORY: List vars(A) and vars(B) separately. If ANY variable appears in both lists, SKIP.
RULE: constant operation

Example: Eq1: (x*y)*z = u*(v*w). vars(LHS)={x,y,z}, vars(RHS)={u,v,w}. Intersection = ∅.
Eq2: x*x = x*(y*y). Both sides are products. VERDICT: TRUE.

2.5 Source-collapse lemmas
If Eq1 is bare (x = P) and P matches one of:
  x*(y*(z*(x*y))), x*((y*z)*(z*z)) -- forces p*q=p (left proj)
  (((y*z)*x)*z)*x -- forces p*q=q (right proj)
Apply projection to Eq2: both sides reduce to same var, VERDICT: TRUE; else VERDICT: FALSE.
RULE: source collapse

2.6 Left projection
If Eq1 is bare: x = x * T.
Step 1: Identify T (everything after x * on the right side).
Step 2: List every variable in T: vars(T) = {___}.
Step 3: Is x in that list? Write: x in T? [YES/NO].
  If YES, SKIP this rule.
  If NO, continue.
Step 4: T has no x, so Eq1 forces a*b=a (left projection).
Step 5: In Eq2, replace every A*B with A (keep leftmost var).
  LHS reduces to: ___. RHS reduces to: ___.
  Same variable? VERDICT: TRUE. Different? VERDICT: FALSE.
RULE: left projection

Example: Eq1: x = x * (y*z). T = y*z. vars(T) = {y,z}. x in T? NO.
Forces a*b=a. Eq2: u*v = v*u. LHS: u. RHS: v. Different. VERDICT: FALSE.

2.7 Right projection
If Eq1 is bare: x = T * x.
Step 1: Identify T (everything before * x on the right side).
Step 2: List every variable in T: vars(T) = {___}.
Step 3: Is x in that list? Write: x in T? [YES/NO].
  If YES, SKIP this rule.
  If NO, continue.
Step 4: T has no x, so Eq1 forces a*b=b (right projection).
Step 5: In Eq2, replace every A*B with B (keep rightmost var).
  LHS reduces to: ___. RHS reduces to: ___.
  Same variable? VERDICT: TRUE. Different? VERDICT: FALSE.
RULE: right projection

Example: Eq1: x = (y*z) * x. T = y*z. vars(T) = {y,z}. x in T? NO.
Forces a*b=b. Eq2: u*v = v*u. LHS: v. RHS: u. Different. VERDICT: FALSE.

2.8 Bare-source contradiction motifs
Apply if Eq1 is bare and no earlier TRUE route fired.
Normalize: single var on LEFT as x, product side = P.
Count rhsVars = number of distinct variables in P.
Check: does x appear as leftmost variable of P? (Lx)
Check: does x appear as rightmost variable of P? (Rx)

Write: rhsVars=_, Lx=_, Rx=_

If rhsVars >= 4 AND Lx=FALSE AND Rx=FALSE, VERDICT: TRUE.
If rhsVars >= 4 AND Lx=FALSE, VERDICT: TRUE.
RULE: contradiction motif

Example: x = y*((z*w)*u). rhsVars=4, Lx=FALSE(y), Rx=FALSE(u). VERDICT: TRUE.

2.9 Direct instance
If Eq2 is obtained from Eq1 by substituting variables (e.g. replacing y with a term throughout), VERDICT: TRUE.
RULE: exact instance

Example: Eq1: x*y = (y*x)*y. Eq2: x*z = (z*x)*z.
Eq2 is Eq1 with y replaced by z throughout. VERDICT: TRUE.

STEP 3. CLEAR SYNTACTIC FALSE ROUTES
Valid only when literal premise is satisfied exactly.

3.0 Spine Isolation
Apply only if Eq1 is bare (x = T) AND x appears exactly once in T.
Step 1: Trace path from root of T to x. At each * node, record L or R.
  Write: path = ___.
Step 2: Classify: All-L = left-spine (depth = count of L). All-R = right-spine. Mixed = SKIP.
  Write: Eq1 type = ___, depth = ___.
Step 3: Do the same for Eq2 (normalize as y = S).
  Write: Eq2 type = ___, depth = ___.
Step 4: Compare:
  Different types? VERDICT: FALSE.
  Same type, Eq2 depth < Eq1 depth? VERDICT: FALSE.
  Same type, Eq1 depth n, Eq2 depth m, n does not divide m? VERDICT: FALSE.
  Otherwise SKIP.
RULE: spine isolation

Example: Eq1 x=((x*y)*z)*w. Path: LLL. Left-spine, depth 3.
Eq2 y=(y*z)*w. Path: LL. Left-spine, depth 2. 3 divides 2? NO. VERDICT: FALSE.

3.1 WORD exclusion
If both sides of Eq1 have the same WORD, but Eq2's sides have different WORD, then VERDICT: FALSE.
RULE: word exclusion

3.2 PARITY exclusion
FIRST check Eq1: compute PARITY (odd/even count for each variable) of both sides. If parities are equal, rule can fire. Else, SKIP.
Example: Eq1: x*(y*x) = (x*y)*x. PARITY LHS: x=even,y=odd. PARITY RHS: x=even,y=odd. Parities are equal, rule can fire.
Example: Eq1: x = (y*x)*(z*(z*x)). PARITY LHS: x=odd. PARITY RHS: x=even,y=odd,z=even. Parities differ. SKIP.
If Eq1 has equal PARITY on both sides AND Eq2 has different PARITY, then VERDICT: FALSE.
RULE: parity exclusion

3.3 Leftmost-variable exclusion
If Eq1 has the same leftmost variable on both sides, but Eq2 does not, then VERDICT: FALSE.
RULE: leftmost exclusion

3.4 Rightmost-variable exclusion
If Eq1 has the same rightmost variable on both sides, but Eq2 does not, then VERDICT: FALSE.
RULE: rightmost exclusion

3.5 SET exclusion
If both sides of Eq1 use the same variable set, but Eq2's sides use different sets, then VERDICT: FALSE.
RULE: set exclusion

3.6 AB (exact count) exclusion
If both sides of Eq1 have the same exact variable counts, but Eq2's sides do not, then VERDICT: FALSE.
RULE: count exclusion

STEP 4. COUNTEREXAMPLE MAGMAS
For each magma below: check Eq1 holds for ALL assignments, then check Eq2 fails for SOME.
If Eq1 holds and Eq2 fails: VERDICT: FALSE.

4.1 Left-zero (a*b=a)
Step 1: Replace every A*B with A recursively until single var.
Step 2: Eq1 LHS reduces to: ___. Eq1 RHS reduces to: ___. Same? If NO, skip.
Step 3: Eq2 LHS reduces to: ___. Eq2 RHS reduces to: ___. Same? If NO, VERDICT: FALSE.
RULE: left-zero witness

4.2 Right-zero (a*b=b)
Step 1: Replace every A*B with B recursively until single var.
Step 2: Eq1 LHS: ___. Eq1 RHS: ___. Same? If NO, skip.
Step 3: Eq2 LHS: ___. Eq2 RHS: ___. Same? If NO, VERDICT: FALSE.
RULE: right-zero witness

4.3 XOR parity (a*b = a+b mod 2)
Step 1: For each variable v, count occurrences in LHS and RHS (mod 2).
Step 2: Eq1: all parities match? If NO, skip.
Step 3: Eq2: all parities match? If NO, VERDICT: FALSE.
RULE: XOR parity witness

4.4 Constant (a*b=c for fixed c)
Step 1: Replace every product with constant c. Bare variables stay.
Step 2: Eq1 holds? (both sides same). If NO, skip.
Step 3: Eq2 holds? If NO, VERDICT: FALSE.
RULE: constant witness

4.5 Left-cycle mod 3 (a*b = (a+1) mod 3)
Step 1: For each side, find the leftmost variable letter.
Step 2: Count left-depth: from root, walk into left child repeatedly until leaf. Count nodes traversed.
Step 3: Eq holds iff both sides have same leftmost variable AND left-depth(LHS) ≡ left-depth(RHS) (mod 3).
Step 4: Eq1 holds? If NO, skip. Eq2 holds? If NO, VERDICT: FALSE.
RULE: left-cycle witness

Example: Eq1: x = ((x*y)*z)*w. Leftmost both sides: x. Left-depth LHS: 0, RHS: 3. 0 mod 3 = 3 mod 3 = 0. Eq1 holds.

4.6 Right-cycle mod 3 (a*b = (b+1) mod 3)
Step 1: For each side, find the rightmost variable letter.
Step 2: Count right-depth: from root, walk into right child repeatedly until leaf. Count nodes traversed.
Step 3: Eq holds iff both sides have same rightmost variable AND right-depth(LHS) ≡ right-depth(RHS) (mod 3).
Step 4: Eq1 holds? If NO, skip. Eq2 holds? If NO, VERDICT: FALSE.
RULE: right-cycle witness

Example: Eq1: x = y*(z*(w*x)). Rightmost both sides: x. Right-depth LHS: 0, RHS: 3. 0 mod 3 = 3 mod 3 = 0. Eq1 holds.

4.7 Left-negation (a*b = 1-a, on {0,1})
Step 1: For each side, find leftmost variable and left-depth.
Step 2: Eq holds iff same leftmost variable AND left-depth(LHS) ≡ left-depth(RHS) (mod 2).
Step 3: Eq1 holds? If NO, skip. Eq2 holds? If NO, VERDICT: FALSE.
RULE: left-negation witness

Step 3: Eq1 holds? If NO, skip. Eq2 holds? If NO, VERDICT: FALSE.
RULE: right-negation witness

4.9 Spike-zero (on {0,1,2}: 0*1=2, all other products=0)
Step 1: Any product of non-bare subterms evaluates to 0. A bare-variable product x*y can be nonzero only when x=0,y=1.
Step 2: Eq1 holds if both sides always evaluate to same value. If one side is bare variable and other is product, Eq1 likely fails. Skip.
Step 3: If Eq1 holds and Eq2 has one bare side and one product side, VERDICT: FALSE.
RULE: spike-zero witness
