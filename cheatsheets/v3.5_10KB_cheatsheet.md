You are solving equational implication problems over magmas: a set with one binary operation * and no other axioms.
Task: decide whether Equation 1 implies Equation 2 over ALL magmas.

Question:
Does {{ equation1 }} imply {{ equation2 }} over all magmas?

OUTPUT FORMAT (mandatory)
First line exactly:
VERDICT: TRUE
or
VERDICT: FALSE
Second line exactly:
RULE: <short rule name>
After that, at most one short sentence.

GLOBAL DISCIPLINE
- Follow the STEPS in order and stop at the first decisive route.
- Never override an exact route with free-form algebra.
- Never use a FALSE route unless its literal premise is satisfied on the written equations.
- If no exact TRUE route and no exact FALSE route fires, use the final fallback only.
- Use transitivity only through the EXPLICIT chains listed below.
- Prefer finite counterexamples before open-ended algebra.

STEP 0. QUICK META-CHECK
Read both equations as binary trees.
Before firing any route, internally ask:
READ: what are the two sides of Equation 1 and Equation 2?
ANALYZE: which exact invariant, family, or finite witness am I checking?
PLAN: choose one route only.
VERIFY:
1. What exact feature am I checking?

If any answer is unclear, reject the route.

Useful terms:

- rightmost variable = last variable in the full leaf-reading
- WORD(term) = full left-to-right leaf list
- PARITY(term) = odd/even occurrence pattern of each variable
- same WORD / same PARITY means exactly equal on the two sides of one equation

STEP 1. EXACT NORMALIZATION
You may:
- rename variables consistently
- swap the two sides of one equation
- dualize BOTH equations together by reversing every product
Do not otherwise simplify.

STEP 2. PREORDER / TRANSITIVITY STRATEGY
Use transititivity only through these safe chains:

2.1 Maximal and minimal laws
- If Equation 2 is x = x, answer TRUE.
RULE: tautological target
- If Equation 1 is x = x and Equation 2 is not x = x, answer FALSE.
RULE: trivial source
- If Equation 1 is singleton collapse, it implies every Equation 2.
RULE: singleton collapse

2.2 Exact transitive chain
If Equation 1 implies an intermediate law by an explicit family rule below, and that intermediate law implies Equation 2 by exact substitution or exact context instance, answer TRUE.
Both links must be exact.
RULE: transitive exact chain

2.3 Safe family preorder
Only these family moves are allowed:
- singleton collapse => any target
- constant operation => any target built only from products, and any target variable = product
- exact renaming / exact substitution / exact context instance => target
No other family preorder is allowed.

STEP 3. IMMEDIATE EXACT TRUE ROUTES

3.1 Exact identity
If Equation 2 is identical to Equation 1 up to consistent variable renaming, answer TRUE.
RULE: exact renaming

3.2 Singleton / collapse source
If Equation 1 has the form x = T where x does not appear anywhere in T, then Equation 1 forces a singleton magma, so every equation holds. Side-swapped form T = x also counts.
Examples: x=y, x=y*z, x=(y*z)*w.
RULE: singleton collapse

3.3 Constant-operation source
If Equation 1 is A = B where:
- A and B are both non-variable product terms, and
- the variable sets of A and B are disjoint,
then all products are forced to one constant value.
Examples: x*y = z*w, (x*y)*z = u*v.
RULE: constant operation

3.4 Projection family
If Equation 1 has one of these shapes:
- x = x*y
- x = x*(y*z)
- x = (x*y)*x
- x = y*x
- x = (y*z)*x
- x = y*(z*x)
and Equation 2 preserves the same leading x or trailing x pattern while inserting or reshaping the opposite side, answer TRUE.
Examples:
- x=x*y implies x=x*(u*v)
- x=(y*z)*x implies x=((u*v)*w)*x
RULE: projection family

3.5 Square-reduction family
If Equation 1 has one of these forms:
- x*x = x
- x*(x*x) = x
- (x*x)*x = x
then any target obtained by replacing a repeated x-block by x is TRUE.
RULE: square reduction

3.6 Direct exact instance
If Equation 2 is exactly a substitution instance of Equation 1, or exactly a one-hole context instance of Equation 1, answer TRUE.
RULE: exact instance

STEP 4. CLEAR SYNTACTIC FALSE ROUTES
These are exact only when their literal premise is satisfied on the written equations.

4.1 WORD exclusion
If Equation 1 has exactly the same full WORD on both sides, but Equation 2 has different WORD on its two sides, answer FALSE.
Legal example: x*(y*z) = (x*y)*z.
Illegal example: x = y*(z*w).
RULE: word exclusion

4.2 PARITY exclusion
If Equation 1 has exactly the same variable parity pattern on both sides, but Equation 2 has different parity pattern, answer FALSE.
RULE: parity exclusion

4.3 Leftmost-variable exclusion
If Equation 1 has the same leftmost variable on both sides, but Equation 2 has different leftmost variables on its two sides, answer FALSE.
RULE: leftmost exclusion

4.4 Rightmost-variable exclusion
If Equation 1 has the same rightmost variable on both sides, but Equation 2 has different rightmost variables on its two sides, answer FALSE.
RULE: rightmost exclusion

STEP 5. FINITE-FIRST SYSTEMATIC REFUTATION
Prefer finite witnesses before any open-ended proof search.
A counterexample is decisive only if Equation 1 holds for ALL assignments and Equation 2 fails for SOME assignment.

Check witnesses in this order:
(1) left-zero
(2) right-zero
(3) constant-0 / constant-1
(4) XOR on {0,1}
(5) tiny linear/affine witnesses from STEP 6

5.1 Left-zero magma
Operation: a*b = a.
Eq1 holds iff its two sides have the same leftmost variable.
Eq2 fails iff its two sides have different leftmost variables.
If so, answer FALSE.
RULE: left-zero witness

5.2 Right-zero magma
Operation: a*b = b.
Eq1 holds iff its two sides have the same rightmost variable.
Eq2 fails iff its two sides have different rightmost variables.
If so, answer FALSE.
RULE: right-zero witness

5.3 XOR / parity witness on {0,1}
Operation: a*b = a+b mod 2.
If Equation 1 is valid because both sides have the same parity pattern, but Equation 2 fails because parity differs, answer FALSE.
RULE: XOR parity witness

5.4 Constant witness on {0,1}
Operation: a*b = 0 for all a,b, or a*b = 1 for all a,b.
Use only if Equation 1 is valid under that constant operation for all assignments and Equation 2 is not.
RULE: constant witness

STEP 6. TINY LINEAR / AFFINE MAGMAS
Use these only as ADDITIONAL finite witnesses, never as vague heuristics.

6.1 Allowed witness menu
You may test exactly these operations:
- on Z/2Z: x*y = x+y mod 2
- on Z/2Z: x*y = x+y+1 mod 2
- on Z/3Z: x*y = x+y mod 3
- on Z/3Z: x*y = x+2y mod 3
- on Z/3Z: x*y = 2x+y mod 3

6.2 Affine witness rule
If under one listed finite linear/affine operation Equation 1 holds for ALL assignments and Equation 2 fails for SOME assignment, answer FALSE.
RULE: linear affine witness

6.3 Safety rule
If evaluating the chosen linear/affine witness is messy or uncertain, reject it.
