You solve equational implication over magmas (one binary op *, no axioms).
Task: does {{ equation1 }} (=A) imply {{ equation2 }} (=B) over ALL magmas?

OUTPUT (strict — line 1 and line 2 mandatory)

RULE: <short rule name>
Then at most 3 short sentences. Reasoning goes AFTER the two header lines.
If uncertain: VERDICT: FALSE with RULE: default false.

DISCIPLINE
- Execute steps in order. Stop at the first decisive rule.
- Never override a fired rule or verified counterexample.
- Never say "Wait", "However", "But actually", "On second thought".
- If no rule proves TRUE → VERDICT: FALSE, RULE: default false.

───────────────────────────────
STEP 0: QUICK TRUE ROUTES
───────────────────────────────

0.1 TAUTOLOGY: B has form x = x → TRUE. RULE: tautology
0.2 EXACT MATCH: A ≡ B up to consistent variable renaming (+optional side swap) → TRUE. RULE: exact renaming
0.3 SINGLETON: A has form x = T where x ∉ vars(T). Side-swapped T = x counts. → TRUE. RULE: singleton
0.4 CONSTANT-OP: A has form P = Q where P,Q are products with disjoint variable sets → TRUE. RULE: constant op
You may rename variables or dualize BOTH equations (reverse every product) before checking.

───────────────────────────────
STEP 1: CANONICALIZE SOURCE
───────────────────────────────

Is one side of A a single variable (no *)? → A is BARE.
If not bare → skip to STEP 4.

If bare, normalize:
1. Put single variable on LEFT → call it x
2. Keep full brackets on product side P
3. x stays x everywhere in P
4. Rename OTHER vars by first appearance in P → y, z, w, u ...

Write: "A canonical: x = [normalized P]"

Example: m = (((n*p)*m)*p)*m
bare var m on LEFT → x. Product: (((n*p)*x)*p)*x → n→y, p→z
A canonical: x = (((y*z)*x)*z)*x

───────────────────────────────
STEP 2: PROJECTION LEMMAS (bare A only)
───────────────────────────────

LEFT PROJECTION FAMILY:
If canonical A has form x = x * T where x ∉ vars(T) → forces p*q = p.
Matches: x=x*y, x=x*(y*z), x=x*(y*(z*w)), x=x*((y*z)*(w*u)).
Exact forms also: x=x*(y*(z*(x*y))), x=x*((y*z)*(z*z)).

RIGHT PROJECTION FAMILY:
If canonical A has form x = T * x where x ∉ vars(T) → forces p*q = q.
Matches: x=y*x, x=(y*z)*x, x=((y*z)*w)*x.
Exact form also: x=(((y*z)*x)*z)*x.

If projection fires, evaluate B under that projection:
- LEFT: replace every P*Q → P recursively until single var
- RIGHT: replace every P*Q → Q recursively until single var
Both sides of B → same var? TRUE. Different? FALSE.
RULE: projection

NON-BARE CONSTANT-PRODUCT:
If A (non-bare), normalized by first appearance, has LHS=x*y, RHS=U*V where all x's are in V, none in U → every product → constant c. Both sides of B same → TRUE. Else FALSE. RULE: constant product

───────────────────────────────
STEP 3: CONTRADICTION MOTIFS (bare A, STEP 2 didn't fire)
───────────────────────────────

Compute 8 features of canonical product side P.
MANDATORY: Write the full vector BEFORE checking any motif.

Features:
- rhsVars = distinct vars in P
- rhsTotals = sorted occurrence counts, no separators (counts 2,1,1 → "112")
- Lx = leftmost var of P is x?
- Rx = rightmost var of P is x?
- xTop = at top split U*V: x only in U → "left", only in V → "right", both → "both"
- topShape = var*product → "v-m", product*var → "m-v", both products → "m-m"
- xCount = occurrences of x in P
- square = P contains u*u for some variable u?

Write: rhsVars=_ Lx=_ Rx=_ xTop=_ topShape=_ rhsTotals=_ xCount=_ square=_

Example: x = y*((z*x)*z)
P = y*((z*x)*z). Vars: {x,y,z} → rhsVars=3. Counts: x:1,y:1,z:2 → rhsTotals=112.
Leftmost=y ≠ x → Lx=F. Rightmost=z ≠ x → Rx=F.
Top split: y * ((z*x)*z) → x in right only → xTop=right.
y is var, ((z*x)*z) is product → topShape=v-m. xCount=1. No u*u → square=F.
→ rhsVars=3 Lx=F Rx=F xTop=right topShape=v-m rhsTotals=112 xCount=1 square=F

If ANY motif matches → VERDICT: TRUE, RULE: contradiction motif, STOP.

STEP 6: SPINE ISOLATION (bare A only, x once in T)
───────────────────────────────

Apply only if A is bare (x = T) AND x appears exactly ONCE in T.
Path from root of T to x: at each * record L (left child) or R (right child).
- All L → pure left-spine, depth = number of L's
- All R → pure right-spine
- Mixed L and R → skip this step

Normalize B as y = S where y appears once in S. Trace same path.
- Left-spine A → B must be left-spine with depth ≥ A. Else FALSE.
- Right-spine A → B must be right-spine with depth ≥ A. Else FALSE.
- Same spine type: A depth n, B depth m → n must divide m. Else FALSE.
RULE: spine isolation

STEP 7: WITNESS ROUTES
───────────────────────────────

Test each model on A, then B. If A holds for ALL assignments but B fails for SOME → FALSE.

W2 Right-zero (a*b=b): replace P*Q → Q recursively. Same test. RULE: right-zero
W3 XOR (a*b=a+b mod 2): holds iff same parity both sides. A same, B different → FALSE. RULE: XOR witness
W4 Constant (a*b=c): all products → c. A holds, B fails → FALSE. RULE: constant witness
